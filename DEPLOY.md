# Deploying MITS Consulting Hub to Render

Single-blueprint deploy. ~25 minutes end to end.

## Prerequisites

1. Code pushed to a GitHub repo (private is fine).
2. A [Render](https://render.com) account connected to that GitHub org.
3. Gmail App Password ready for the system SMTP account (`vaibhav.aggarwal@mitssolution.com`).
4. Google OAuth Client ID + Secret (you already created this earlier — same one used in dev).

> Cost: ~₹0 for first 90 days (Postgres free tier).
> After: ~$15/mo (Postgres $7 + Backend $7 + Frontend free).

---

## Step 1 — Push the code to GitHub (one time)

From the repo root:

```bash
cd /Users/shivamaggarwal/mits-consulting
git init && git add -A && git commit -m "Initial portal"
gh repo create mits-consulting --private --source=. --push
```

Or via github.com UI: create empty repo → follow the *push existing* instructions.

The repo's `.gitignore` already excludes `.env`, `node_modules`, `dist`, and `uploads/`.

---

## Step 2 — Create the Render blueprint

1. Open https://dashboard.render.com → **New +** → **Blueprint**.
2. Connect your GitHub account if not already.
3. Pick the `mits-consulting` repo.
4. Render reads `/render.yaml` and previews **3 resources** that will be created:
   - Postgres database: `mits-postgres`
   - Web service:       `mits-backend`
   - Static site:       `mits-frontend`
5. Click **Apply**. Render provisions the DB first (~2 min), then builds the backend (~4 min), then the frontend (~2 min).

You don't need to set anything yet — wait for the build to finish (it will fail on first run because `sync: false` env vars are blank; that's expected, fix them next).

## Step 3 — Set the deferred env vars

Render flagged some env vars as `sync: false` (intentional — they're secrets / URLs).

### Backend (`mits-backend` → Environment tab)

| Key | Value |
|---|---|
| `CLIENT_ORIGIN`         | `https://mits-frontend.onrender.com` *(swap to the actual frontend URL shown in Render dashboard)* |
| `SMTP_USER`             | `vaibhav.aggarwal@mitssolution.com` |
| `SMTP_PASS`             | *(your 16-char Gmail App Password, no spaces — from local `.env` / 1Password)* |
| `SMTP_FROM`             | `"MITS Consulting Portal <vaibhav.aggarwal@mitssolution.com>"` |
| `GOOGLE_CLIENT_ID`      | *(from local `.env` — Google Cloud Console → APIs & Services → Credentials)* |
| `GOOGLE_CLIENT_SECRET`  | *(from local `.env` — same OAuth client)* |
| `GOOGLE_REDIRECT_URI`   | `https://mits-backend.onrender.com/api/oauth/google/callback` *(swap to actual backend URL)* |

`JWT_SECRET` and `SMTP_USER_ENCRYPTION_KEY` were auto-generated by Render. Leave them alone.

### Frontend (`mits-frontend` → Environment tab)

| Key | Value |
|---|---|
| `VITE_API_URL` | `https://mits-backend.onrender.com` *(swap to actual backend URL)* |

After saving env vars on both services, click **Manual Deploy → Clear build cache & deploy** on each so the new values take effect.

## Step 4 — Update Google OAuth allowed URIs

In https://console.cloud.google.com → APIs & Services → Credentials → click your OAuth client:

**Authorized JavaScript origins** — add:
```
https://mits-frontend.onrender.com
https://mits-backend.onrender.com
```
(Keep the localhost ones for dev.)

**Authorized redirect URIs** — add:
```
https://mits-backend.onrender.com/api/oauth/google/callback
```

Save. New URI applies in ~30 sec.

## Step 5 — Run migrations + seed test data

The `buildCommand` already runs `prisma migrate deploy`, so your schema is in place. You still need data.

In Render dashboard → `mits-backend` → **Shell** tab → click **Start** → wait for the prompt → run:

```bash
npm run seed          # creates users, banks, lead sources, templates, flags
npm run seed:stress   # 1500 trainers, 678 clients (40 demos / 100 dormant / 300 hold)
```

Takes ~3 minutes. You'll see the same summary you saw locally.

> If you want lighter data first: run `npm run seed:big` instead (80 clients, 26 trainers).

## Step 7 — Custom domain (optional, 10 min)

1. Buy/own `mitssolution.com` (or a subdomain).
2. Render → `mits-frontend` → **Settings** → **Custom Domain** → add `portal.mitssolution.com`.
3. Render shows a CNAME target — add it at your DNS provider:
   ```
   portal  CNAME  mits-frontend.onrender.com
   ```
4. SSL is automatic.
5. Update env vars to the new domain:
   - Backend `CLIENT_ORIGIN` → `https://portal.mitssolution.com`
   - Frontend (no change — you can leave the onrender URL or also alias the backend)
6. Update Google OAuth allowed origins/redirect URIs to include the new domain.

## Common pitfalls

| Issue | Fix |
|---|---|
| Backend build fails on `prisma migrate deploy` | Postgres URL not set, or DB still provisioning — redeploy after 2 min |
| Login redirect loop / cookie not set | `CLIENT_ORIGIN` doesn't exactly match the frontend URL → CORS rejects → cookie never lands. Match the URL exactly, including https + no trailing slash. |
| Google sign-in errors `redirect_uri_mismatch` | Your `GOOGLE_REDIRECT_URI` env var doesn't match what's in Google Console. They must be identical character-for-character. |
| SMTP error 535-5.7.8 | Wrong App Password, or 2FA not enabled on the account. Regenerate from myaccount.google.com/apppasswords. |
| First request after idle is slow | Render free tier sleeps backends after 15 min idle. Upgrade to Starter ($7/mo) to disable. |
